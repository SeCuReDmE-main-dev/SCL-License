# Contributing To The SecuredMe License Family

This repository maintains the SEL/SECL license family:

- `LicenseRef-SEL-2.0`: Secured Educational License 2.0.
- `LicenseRef-SECL-2.0`: Secured Educational Cybersecurity License 2.0.

Contributions must keep the license family clear, bounded, educational, and
misuse-aware.

## Accepted Contributions

- typo fixes and clarity improvements;
- safer attribution, NOTICE, DISCLAIMER, or SAFETY wording;
- examples showing how to apply `LicenseRef-SEL-2.0` or
  `LicenseRef-SECL-2.0`;
- adoption checklist improvements;
- documentation that reduces ambiguity for schools, teachers, students, and
  supervised young adults.

## Not Accepted

- claims that SEL/SECL are OSI-approved or SPDX-listed before that is true;
- attack, theft, fraud, bypass, unsafe surveillance, or criminal automation
  enablement;
- removal of the no-warranty, no-authority, no-misuse-responsibility boundary;
- changes that make OpenAI/Codex, Google/Gemini/Antigravity, schools, reviewers,
  or SecuredMe responsible for unsafe private forks.

## Review Path

All official changes must go through pull request review. Private forks are
allowed according to the selected license text, but they are not the maintained
school version unless accepted by maintainers.
