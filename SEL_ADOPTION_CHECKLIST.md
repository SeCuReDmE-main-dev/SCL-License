# SEL / SECL Adoption Checklist

Before applying `LicenseRef-SEL-2.0` or `LicenseRef-SECL-2.0` to a repository:

- [ ] The project is educational, research-oriented, simulation-oriented, or supervised training-oriented.
- [ ] Cybersecurity, fraud-awareness, safety, and abuse-prevention projects use `LicenseRef-SECL-2.0`.
- [ ] General education projects use `LicenseRef-SEL-2.0`.
- [ ] `LICENSE` contains the full SEL-2.0 text.
- [ ] `NOTICE` contains maintainer attribution and ORCID.
- [ ] `DISCLAIMER` is present.
- [ ] `SAFETY.md` forbids attack, theft, fraud, bypass, unsafe surveillance, and criminal automation.
- [ ] The README states the project is educational and human-review-only where relevant.
- [ ] Existing licenses were not replaced without explicit approval.
- [ ] AI provider language does not add unsupported classroom routes.
- [ ] Private modified copies are not presented as the official maintained school version.
- [ ] Tests or smoke checks are documented before release.
